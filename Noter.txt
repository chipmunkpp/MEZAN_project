================================================================
  VERSION 0.3  —  Update v0.3
================================================================
Date   : 2026-04-17
Status : Released
Branch : from v0.2 final

Changes:
  1. STICKY NOTE REACTION BUTTON FIX  (bug)
     - Emote buttons (👍 ❤️ 😂 😮) on sticky notes were unclickable.
     - Root cause: rendered markup was
         onclick="toggleReact(<id>,"👍")"
       The JSON-stringified emoji injected a literal " inside a
       double-quoted attribute, closing the onclick early so the
       handler silently never fired.
     - Fix: switched the onclick attribute wrapper to single quotes
       in renderNote() and the toggleReact() re-render path, so the
       inner "…" from JSON.stringify is preserved correctly.
     - Files: index.html (renderNote + toggleReact).

  2. RECURRING MANAGEMENT  (v0.3 Priority-2 feature from roadmap)
     Data model
       - New localStorage key `sv98_recurring` with DB helpers:
         getRecurrings / saveRecurrings / addRecurring / patchRecurring
         / delRecurring.
       - Each rule stores:
         { id, description, amount, category, type, account,
           interval (daily|weekly|monthly), startDate, lastPosted,
           active }.
       - Key added to the Electron saveAllData payload and the boot
         rehydrate path → survives restart and syncs to %APPDATA%.
     Creation flow
       - Add Transaction form unchanged from the user's POV: picking
         Daily/Weekly/Monthly now creates a persistent rule instead
         of cloning the next 3 instances up-front.
       - The submitted transaction itself counts as the first post;
         lastPosted = date.
     Auto-post on boot
       - New postOverdueRecurrings() walks every active rule and
         posts one txn per missed interval from
           advanceDate(lastPosted, interval) … today
         Each posted txn is stamped with recurringRuleId.
       - Runs inside boot() right after the IndexedDB image migration.
     Management UI
       - New "🔁 Recurring Rules" panel in Settings window.
       - Rows show: type arrow, description + category/account,
         amount, interval, last-posted date, active/paused badge.
       - Per-row actions: Pause/Resume (toggleRecurring), ✕ Delete
         (deleteRecurring — confirm dialog; already-posted txns are
         kept).
       - Panel is refreshed automatically by renderSettings() and
         after creating a new rule from Add Transaction.

  3. VERSION BUMP
     - package.json           : 0.2.0 → 0.3.0
     - About dialog           : "Version 0.2" → "Version 0.3"
     - Noter.txt              : this section added

  VERSION 0.4  —  Update v0.4.1  (Business / Freelancer Mode)
================================================================
Date   : 2026-04-17
Status : Released
Branch : from v0.3

Overview
  Expands Saver 98's use case beyond personal finance into small-
  business and freelance scenarios. You can now carve out money
  tracking per client, studio, or engagement — each with its own
  sub-projects, documents, transactions, and budget — without
  mixing into the main household ledger.

Feature: PROJECTS
  Data model (new localStorage key `sv98_projects`, synced to disk):
    project       : { id, name, description, createdDate,
                      subProjects: [...] }
    subProject    : { id, name, description, createdDate,
                      documents: [...],
                      transactions: [...],
                      budget: { total, categories } }
    document      : { id, date, type (invoice|receipt|payment|
                      contract|other), title, amount, note }
    transaction   : { id, date, type (income=Received |
                      expense=Paid), description, category,
                      amount, note }

  UI
    • New "📁 Projects" desktop icon (top:688px) + Start Menu entry.
    • Single resizable window `win-projects` (820×620, min 560×420)
      with breadcrumb navigation and internal view state:
        list → project → sub-project
    • "List" view: card grid of all projects with KPI chips
      (received / paid / net / sub-count / txn-count).

  Project view (tabs)
    📊 Dashboard     — KPI cards + recent activity across sub-projects
                       (clickable to jump into the sub-project)
    📂 Sub-Projects  — card grid; "+ New Sub-Project"; deep-link
    💰 Budget        — overall spend bar + per-sub-project budget
                       table with % used (blue/orange/red)

  Sub-Project view (tabs)
    📋 Overview      — description + KPI cards + created date
    📄 Documents     — list (date, type, title, amount, note) +
                       inline add form (type dropdown covers
                       invoice / receipt / payment / contract /
                       other); delete per row
    💵 Paid/Received — list + inline add form (Received = income,
                       Paid = expense); delete per row
    💰 Budget        — set total budget + progress bar + remaining

  CRUD helpers
    Proj.promptNewProject / editProject / deleteProject
    Proj.promptNewSub     / editSub     / deleteSub
    Proj.addDoc           / deleteDoc
    Proj.addTxn           / deleteTxn
    Proj.saveSubBudget

  Persistence
    • DB.getProjects / saveProjects
    • Electron file-sync payload extended with DB.PK
    • Boot hydrate reads DB.PK from %APPDATA%
    • Kept isolated from the main sv98_txn ledger — Project
      transactions do NOT affect the Dashboard, History, Charts,
      or main Budget warnings.

  Helper: showInput(title, label, initial)
    • Minimal Win98-style modal input prompt used for project /
      sub-project naming and editing (keyboard: Enter = OK,
      Escape = Cancel).

Version bump
  - package.json   : 0.3.0 → 0.4.0
  - About dialog   : "Version 0.3" → "Version 0.4"

v0.4 POLISH PATCHES (2026-04-17)
  A. Desktop icons no longer clip off-screen when the Electron

  B. Charts window: new "Projects" tab. drawProjects(cv) renders

v0.4 PATCH 2 — 2026-04-17 (local-only, no cloud API)
  1. Project / sub-project create flow consolidated: a single
     modal dialog (showProjectForm) collects name + description
     together instead of two sequential prompts. Reused by edit.

  2. Sticky note "Archive" + Gallery redesign:
     • New 🗃️ button on each sticky note title bar calls
       archiveNote(id) — moves the note out of the live desktop
       into DB.AK (`sv98_archived_notes`).
     • "Images" desktop icon renamed to "🖼️ Gallery".
       Gallery window now has a Win98 tab bar with two sections:
         – Images  : slips/receipts (existing behavior)
         – Archived: grid of saved sticky notes with
                     Restore / Delete actions per card.
     • restoreArchivedNote / deleteArchivedNote helpers.

  3. User Profiles (local-only):
     • New storage keys
         DB.PRK = sv98_profiles        (list of profiles)
         DB.APK = sv98_active_profile  (active id)
     • PER_PROFILE_KEYS covers: transactions, settings/budgets,
       sticky notes, recurring rules, projects, archived notes.
       The profile system keys themselves are NOT per-profile
       (so switching doesn't wipe the profile list).
     • snapshotLiveProfile()   — freeze current localStorage into
                                 the active profile's `data` slot.
     • applyProfileSnapshot()  — hydrate localStorage from a
                                 profile's frozen snapshot.
     • ensureProfileSystem()   — first-run wrap: if no profiles
                                 exist, create "Default" wrapping
                                 existing data. Called in boot()
                                 after hydrate, before init().
     • switchProfile / addProfile / deleteProfile / renameProfile
     • softReloadProfile()     — closes floating stickies, then
                                 refreshes all views after switch.
     • Taskbar: 👤 profile indicator chip between Start and
       window list. Click → opens Settings scrolled to the
       profile panel. updateProfileIndicator() runs on init
       and after any profile mutation.
     • Settings: "👤 User Profiles" panel above Data Management.
       Lists profiles with Switch / Rename / Delete, plus
       "+ New Profile" and "💾 Save Current" buttons.
       Rendered by renderProfilePanel(), called from
     renderSettings().
     • Electron file-sync payload extended with DB.PRK + DB.APK;
       boot hydrate reads both. Entire system is local-file /
       localStorage — no cloud API.
